# XIAO ESP32S3 全模块综合测试程序

## 📋 功能说明

这是一个综合测试固件，用于验证所有模块的功能。

---

## 🎬 新功能：动画播放

支持从8MB Flash读取JPEG序列帧动画并播放到屏幕上。使用PSRAM进行缓存。

### 动画文件说明
| 文件夹 | 帧数 | 文件名格式 |
|--------|------|-----------|
| `anim1/` | 182帧 | 0001.jpg - 0182.jpg |
| `anim2/` | 35帧 | 0001.jpg - 0035.jpg |
| `anim3/` | 166帧 | 0001.jpg - 0166.jpg |

### 动画播放命令

| 命令 | 功能 |
|------|------|
| `a1` | 播放动画1 (182帧) |
| `a2` | 播放动画2 (35帧) |
| `a3` | 播放动画3 (166帧) |
| `a0` | 停止动画 |
| `al` | 切换循环模式 |

### ⚡ 上传动画数据到Flash

#### 方法1：使用Arduino IDE LittleFS插件（推荐）

1. **安装ESP32 LittleFS上传插件**
   - 下载插件：https://github.com/lorol/arduino-esp32littlefs-plugin/releases
   - 解压后将 `ESP32LittleFS` 文件夹放入 `Arduino/tools/` 目录
   - 最终路径应为：`Arduino/tools/ESP32LittleFS/tool/esp32littlefs.jar`
   - 重启Arduino IDE

2. **配置分区表**
   - Arduino IDE -> 工具 -> Partition Scheme
   - 选择 **"8M with spiffs (3MB APP/1.5MB SPIFFS)"**
   - 或选择自定义分区以获得更大存储空间

3. **准备data文件夹**
   - 确保 `data` 文件夹与 `test_all_modules.ino` 在同一目录
   ```
   test_all_modules/
   ├── test_all_modules.ino
   └── data/
       ├── anim1/
       │   ├── 0001.jpg
       │   ├── 0002.jpg
       │   └── ...
       ├── anim2/
       │   └── ...
       └── anim3/
           └── ...
   ```

4. **上传data文件夹**
   - Arduino IDE -> 工具 -> ESP32 Sketch Data Upload
   - 等待上传完成（可能需要几分钟）

#### 方法2：使用 esptool.py 命令行

```bash
# 1. 安装 mklittlefs 工具
# Windows: 从 https://github.com/earlephilhower/mklittlefs/releases 下载

# 2. 创建LittleFS镜像 (根据分区大小调整 -s 参数)
mklittlefs -c data -b 4096 -p 256 -s 3145728 littlefs.bin

# 3. 上传到ESP32S3 (根据实际COM端口和分区偏移地址修改)
esptool.py --chip esp32s3 --port COM3 --baud 921600 write_flash 0x290000 littlefs.bin
```

#### 方法3：使用 PlatformIO

在 `platformio.ini` 中配置：
```ini
[env:esp32s3]
platform = espressif32
board = seeed_xiao_esp32s3
framework = arduino
board_build.filesystem = littlefs
board_build.partitions = default_8MB.csv

# 启用PSRAM
build_flags = 
    -DBOARD_HAS_PSRAM
    -mfix-esp32-psram-cache-issue
```

然后运行：
```bash
pio run --target uploadfs
```

### 📚 需要安装的库

动画播放功能需要额外安装 **JPEGDEC** 库：
- Arduino IDE -> 工具 -> 管理库
- 搜索 `JPEGDEC` (作者: Larry Bank)
- 点击安装

### 🖼️ 图片格式要求

- 格式：JPEG
- 分辨率：建议 170x320 (与屏幕匹配) 或更小
- 单张大小：建议 < 60KB
- 命名：4位数字，如 0001.jpg, 0002.jpg

## 🔌 硬件连接

### 完整接线表

```
模块          引脚       ESP32S3
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ST7789        CS      -> D0
屏幕          DC      -> D1
              RST     -> D2
              SDA     -> D10
              SCL     -> D8
              BL      -> VCC
              VCC     -> 3.3V
              GND     -> GND

PCA9685       SDA     -> D4
(I2C)         SCL     -> D5
              VCC     -> 3.3V或5V
              V+      -> 外部6V (舵机电源)
              GND     -> GND

MAX98357A     BCLK    -> D11 (底部焊盘)
(I2S音频)     LRC     -> D12 (底部焊盘)
              DIN     -> D9
              VCC     -> 5V
              GND     -> GND

URT-1         TX      -> D6
(总线舵机)    RX      -> D7
              VCC     -> 外部6-12V
              GND     -> GND
```

### ⚠️ 重要提示

1. **底部焊盘 D11/D12**
   - 需要在电路板背面焊接引线
   - 这两个引脚对应 GPIO42/GPIO41
   - 不与摄像头冲突

2. **电源共地**
   - 所有模块的 GND 必须连接在一起
   - PCA9685 的 V+ 使用外部电源（舵机供电）
   - URT-1 使用外部电源（总线舵机供电）

3. **URT-1 舵机 ID**
   - 测试前需将三个总线舵机 ID 分别设置为 **1、2、3**
   - 或修改代码中的 ID 号

## 🚀 使用方法

### 1. 上传程序

```bash
1. Arduino IDE 中打开 test_all_modules.ino
2. 选择板子: XIAO_ESP32S3
3. 选择端口
4. 点击上传
```

### 2. 打开串口监视器

```

```

### 3. 测试命令

在串口监视器中输入数字：

| 命令 | 功能 | 说明 |
|------|------|------|
| **1** | 屏幕测试 | 红→绿→蓝循环，每秒切换 |
| **2** | PCA9685测试 | 0和1号舵机交替旋转 |
| **3** | 音频测试 | 播放三个音调 (A-C-E) |
| **4** | URT-1测试 | 1/2/3号舵机同时来回旋转 |
| **5** | 全部测试 | 所有功能同时运行 |
| **0** | 停止 | 停止所有测试 |

## 🔧 故障排除

### 问题1: 屏幕不显示

**检查:**
- CS, DC, RST, SDA, SCL 连接是否正确
- VCC 是否有 3.3V
- BL 背光是否接 VCC

**解决:**
- 参考之前的 test_basic.ino 测试

### 问题2: PCA9685 舵机不动

**检查:**
- I2C 连接 (SDA-D4, SCL-D5)
- V+ 外部电源是否连接（6V）
- 舵机信号线是否接对通道

**测试:**
- 串口应显示 "✓ PCA9685初始化完成"
- 舵机应该有"哒哒"声

### 问题3: 音频无声

**检查:**
- D11/D12 底部引脚是否焊接
- MAX98357A 电源是否正常
- 扬声器是否连接

**常见原因:**
- D11/D12 未焊接或接错
- GPIO编号不对（试试改为 42/41）

**修改代码测试:**
```cpp
// 如果没声音，尝试交换引脚
#define I2S_BCLK  41  // 改为41
#define I2S_LRC   42  // 改为42
```

### 问题4: URT-1 舵机不响应

**检查:**
- TX/RX 是否接反（常见错误）
- 舵机 ID 是否正确（必须是123）
- 舵机电源是否充足
- 波特率是否匹配（115200）

**测试舵机ID:**
- 使用 URT-1 配套的调试软件
- 确认舵机ID并修改代码

### 问题5: 编译错误

**缺少库:**
```
需要安装以下库：
- Adafruit GFX Library
- Adafruit ST7789
- Adafruit PWM Servo Driver Library
```

**安装方法:**
- Arduino IDE -> Tools -> Manage Libraries
- 搜索库名并安装
