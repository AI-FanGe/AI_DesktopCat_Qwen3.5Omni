# ESP32S3 8MB Flash 分区表配置

## 问题原因

ESP32S3 虽然有 8MB Flash，但默认分区表分配给文件系统的空间只有 ~1.5MB。

## 自定义分区表

`partitions_custom.csv` 文件分配方案：

| 分区名 | 类型 | 大小 | 说明 |
|--------|------|------|------|
| nvs | data | 20KB | 存储配置 |
| otadata | data | 8KB | OTA数据 |
| app0 | app | **3MB** | 程序代码 |
| spiffs | data | **~5MB** | LittleFS文件系统 |

**总计：8MB**（去掉保留区域）

## 配置方法

### 方法1：Arduino IDE 内置选项（推荐）

1. 打开 Arduino IDE
2. **工具 → Partition Scheme**
3. 选择以下任一选项：
   - `8M with spiffs (3MB APP/1.5MB SPIFFS)` - 默认
   - `Huge APP (3MB No OTA/1MB SPIFFS)` - 程序大，文件系统小
   - `Minimal SPIFFS (1.9MB APP with OTA/190KB SPIFFS)` - 最小文件系统
   - **`No OTA (Large APP)` 或类似选项** - 如果不需要OTA更新

### 方法2：使用自定义分区表

1. 将 `partitions_custom.csv` 放在 sketch 目录下（已完成）
2. Arduino IDE → **工具 → Partition Scheme → Custom**
3. 重新编译上传

如果没有 "Custom" 选项，需要手动添加到 boards.txt 配置文件。

### 方法3：直接修改 boards.txt

1. 找到 ESP32 核心文件夹：
   ```
   C:\Users\用户名\AppData\Local\Arduino15\packages\esp32\hardware\esp32\版本号\
   ```

2. 编辑 `boards.txt`，找到 `xiao_esp32s3` 相关配置，添加：
   ```
   xiao_esp32s3.menu.PartitionScheme.custom=Custom (partitions.csv)
   xiao_esp32s3.menu.PartitionScheme.custom.build.partitions=partitions
   xiao_esp32s3.menu.PartitionScheme.custom.upload.maximum_size=3145728
   ```

3. 重启 Arduino IDE

### 方法4：使用 PlatformIO（最灵活）

在 `platformio.ini` 中：
```ini
[env:esp32s3]
platform = espressif32
board = seeed_xiao_esp32s3
framework = arduino
board_build.partitions = partitions_custom.csv
board_build.filesystem = littlefs
```

## 验证配置

烧录 `littlefs_uploader.ino` 后，串口监视器会显示：

**修改前（默认分区）：**
```
[ 存储信息 ]
  总空间: 1408 KB (1.4 MB)    ← 只有1.4MB
```

**修改后（自定义分区）：**
```
[ 存储信息 ]
  总空间: 4864 KB (4.8 MB)    ← 约5MB可用
```

## 常见分区方案对比

| 方案名称 | App大小 | 文件系统 | 适用场景 |
|---------|---------|----------|---------|
| Default 4MB | 1.2MB | 1.5MB | 小程序 |
| 8M with spiffs | 3MB | 1.5MB | 默认8MB |
| No OTA | 3.9MB | 1.5MB | 不需要OTA |
| **Custom (本项目)** | 3MB | **~5MB** | 大量动画文件 |

---

## 注意事项

1. 修改分区表后，**Flash中的所有数据会丢失**
2. 程序区 3MB 对于大多数项目足够
3. 如果程序编译后超过 3MB，需要调整分区大小
